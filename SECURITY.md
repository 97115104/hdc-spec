# Reporting security issues in AIISP-1

If you believe you have found a security issue in the AIISP-1 specification, the reference implementation, or the on-chain settlement contract once published, please report it privately to the editor at `x@97115104.com` rather than filing a public issue.

Please include:

- A description of the issue.
- Reproduction steps where applicable.
- Your assessment of the impact.
- Any suggested mitigation.

The editor commits to acknowledging the report within 72 hours and to publishing a coordinated disclosure once a fix or mitigation is available. Reporters will be credited in the spec's revision history unless they request otherwise.

For non-security errata (typos, broken links, arithmetic errors), please use the public Errata issue template instead.
