1. What this is

AIISP-1 is an open standard for breaking the dollar amount you pay for a single Large Language Model (LLM) inference request into named, machine-readable parts: how much was electricity, how much was water and carbon, how much was the provider's premium, and how much is owed to the people whose work made the model possible.

It is a small specification — two optional request headers, two response headers, one JSON record, one settlement contract. It is opt-in, transport-agnostic, and designed to be added to any existing HTTP-based inference API in an afternoon.

It is published in the spirit of the early internet RFCs: short, composable, opt-in, and owned by no single party.

2. Why it is useful

Today, when a model answers a question, exactly one number is exposed to the requester: the price. That number bundles together electricity, cooling water, embodied carbon of the chips, amortized cost of the training data, and the provider's margin. Because the components are bundled, none of them can be priced, audited, or compensated separately.

AIISP-1 unbundles them on the wire, in a format any auditor, regulator, or downstream pipeline can read, and settles the unbundled amounts on a public ledger so the receiving parties — environmental funds, contributor pools, peer reviewers — actually get paid in the same transaction that bills the client.

5. Who this is for

  • Inference providers — you can implement AIISP-1 in an afternoon and gain a public, auditable record of where each dollar of compute revenue goes.
  • Regulators — AIISP-1 produces the machine-readable per-request disclosure that recent state and federal AI legislation is moving toward, without requiring a new agency or a compute threshold.
  • Contributors and creators — the attested-author pool is the channel by which the people whose work trained these models can actually be paid, on the same transaction that bills the user of the model.
  • Communities bearing environmental cost — the carbon and water shares are routed to dedicated funds, with the explicit goal of moving from restoration-certificate proxies to direct grants in the affected watershed as fast as the on-chain primitives allow.
  • Auditors and academic researchers — every settled request is a public on-chain record. Sample, recompute against the published energy rate, compare. The arithmetic is small, the data is open.

7. Status and disclosure

AIISP-1 v0.1 is a draft. It is not a ratified standard of any standards-development organization. The editor intends to submit it for community review through the IETF ART area, the Decentralized Identity Foundation, and the W3C AI Provenance community group.

The editor (Austin Harshberger) is the principal of Happy Stack Calculus LLC, which intends to operate the reference implementation of AIISP-1. This is a present and ongoing conflict of interest. The plan, documented in GOVERNANCE.md, is to migrate ratification authority for v1.0 to a multi-stakeholder body before AIISP-1 is declared stable. The editor's compensation, once any settlement flows exist, will be published on the same on-chain ledger as contributor compensation.
